01 PROJECT OVERVIEW [cite: 10]
The platform is a subscription-driven web application combining golf performance tracking, charity fundraising, and a monthly draw-based reward engine. [cite: 11] It is designed to feel emotionally engaging and modern - deliberately avoiding the aesthetics of a traditional golf website. [cite: 12] Users will: [cite: 13]
• Subscribe to the platform (monthly or yearly) [cite: 14]
• Enter their latest golf scores in Stableford format [cite: 14]
• Participate in monthly draw-based prize pools [cite: 14]
• Support a charity of their choice with a portion of their subscription [cite: 15]

02 CORE OBJECTIVES [cite: 16]
• Subscription Engine: Build a robust subscription and payment system [cite: 17]
• Score Experience: Simple, engaging score-entry flow [cite: 17]
• Custom Draw Engine: Algorithm-powered or random monthly draws [cite: 17]
• Charity Integration: Seamless charity contribution logic [cite: 17]
• Admin Control: Comprehensive admin dashboard and tools [cite: 17]
• Outstanding UI/UX: Design that stands out in the golf industry [cite: 17]

04 SUBSCRIPTION & PAYMENT SYSTEM [cite: 40]
• Plans: Monthly plan and Yearly plan (discounted rate) [cite: 41]
• Gateway: Stripe (or equivalent PCI-compliant provider) [cite: 41]
• Access Control: Non-subscribers receive restricted access to platform features [cite: 41]
• Lifecycle: Handles renewal, cancellation, and lapsed-subscription states [cite: 41]
• Validation: Real-time subscription status check on every authenticated request [cite: 41]

05 SCORE MANAGEMENT SYSTEM [cite: 42]
Input Requirements [cite: 43]
• Users must enter their last 5 golf scores [cite: 44]
• Score range: 1-45 (Stableford format) [cite: 45]
• Each score must include a date [cite: 46]

Functional Behaviour [cite: 47]
• Only the latest 5 scores are retained at any time [cite: 48]
• A new score replaces the oldest stored score automatically [cite: 49]
• Scores are displayed in reverse chronological order (most recent first) [cite: 50]

06 DRAW & REWARD SYSTEM [cite: 51]
Draw Types [cite: 52]
• 5-Number Match [cite: 53]
• 4-Number Match [cite: 54]
• 3-Number Match [cite: 55]

Draw Logic Options [cite: 56]
• Random generation standard lottery-style draw [cite: 57, 58]
• Algorithmic - weighted by most/least frequent user scores [cite: 59]

Operational Requirements [cite: 60]
• Monthly cadence draws executed once per month [cite: 61]
• Admin controls publishing of draw results [cite: 62]
• Simulation/pre-analysis mode before official publish [cite: 63]
• Jackpot rollover to next month if no 5-match winner [cite: 63]

07 PRIZE POOL LOGIC [cite: 68]
A fixed portion of each subscription contributes to the prize pool. [cite: 69] Distribution is pre-defined and enforced automatically. [cite: 69]
• 5-Number Match: 40% | Yes (Jackpot) [cite: 70]
• 4-Number Match: 35% | No [cite: 70]
• 3-Number Match: 25% | No [cite: 70]
• Auto-calculation of each pool tier based on active subscriber count [cite: 71]
• Prizes split equally among multiple winners in the same tier [cite: 72]
• 5-Match jackpot carries forward if unclaimed [cite: 73]

08 CHARITY SYSTEM [cite: 74]
Contribution Model [cite: 75]
• Users select a charity at signup [cite: 76]
• Minimum contribution: 10% of subscription fee [cite: 77]
• Users may voluntarily increase their charity percentage [cite: 78]
• Independent donation option (not tied to gameplay) [cite: 79]

Charity Directory Features [cite: 80]
• Charity listing page with search and filter [cite: 81]
• Individual charity profiles: description, images, upcoming events (e.g. golf days) [cite: 82]
• Featured/spotlight charity section on homepage [cite: 83]

09 WINNER VERIFICATION SYSTEM [cite: 84]
• Eligibility: Verification process applies to winners only [cite: 85]
• Proof Upload: Screenshot of scores from the golf platform [cite: 85]
• Admin Review: Approve or Reject submission [cite: 85]
• Payment States: Pending → Paid [cite: 85]

12 UI/UX REQUIREMENTS [cite: 119]
The platform must not resemble a traditional golf website. [cite: 120] Design must be emotion-driven - leading with charitable impact, not sport. [cite: 120]
• Feel: Clean, modern, motion-enhanced interface [cite: 121]
• Avoid: Golf clichés (fairways, plaid, club imagery as primary design language) [cite: 121]
• Homepage: Clearly communicates: what the user does, how they win, charity impact, and CTA [cite: 121]
• Animations: Subtle transitions and micro-interactions throughout [cite: 121]
• СТА: Subscribe button / flow must be prominent and persuasive [cite: 121]

• Fast performance optimised assets, minimal blocking resources JWT or session-based, HTTPS enforced [cite: 124]
• Secure authentication [cite: 125]
